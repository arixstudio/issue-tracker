Issue Tracker Application API

API documentation for an Issue Tracker application to create, read, update, and delete issues. The authentication endpoint for Google OAuth returns an HTML page requiring user interaction. This design choice is intended for web application flows where user involvement is feasible and expected. Ensure client compatibility and consider the user experience and security implications of this approach. Authentication relies on cookies set by the server. Created by Team 8 [FED25].

get /auth/google
Redirect to Google for authentication (authGoogleGet)
Initiates authentication with Google. Users are presented with an HTML page that requires interaction - specifically, clicking on a Google sign-in button. This method is suitable for scenarios where user interaction can be managed within a web application context. Ensure your implementation adheres to OAuth and security best practices, considering both the user experience and potential security implications.

Return type

String

Example data

Content-Type: application/json
"
Please click on the Google sign-in button to authenticate.
"

Produces

This API call produces the following media types according to the Accept request header; the media type will be conveyed by the Content-Type response header.
  • text/html
